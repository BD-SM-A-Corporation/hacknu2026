# API Документация (v1)

Все API эндпоинты проекта **RailTelemetry** находятся под префиксом `/api/` и требуют авторизации (`auth:sanctum`), за исключением публичных. Для авторизации необходимо отправлять токен `Bearer` в заголовке `Authorization`, либо использовать файлы cookie при работе из встроенного SPA (InertiaJS).

## Локомотивы (Locomotives)

Управление справочником локомотивов и их метриками.

### 1. Получить список локомотивов
**GET** `/api/locomotives`
Возвращает список всех зарегистрированных локомотивов.

### 2. Добавить локомотив
**POST** `/api/locomotives`
- `id` (string, required): Уникальный идентификатор (например, `KZ8A-001`)
- `model` (string, required): Модель (серия) локомотива
- `depot` (string, optional): Депо приписки

### 3. Получить информацию о локомотиве
**GET** `/api/locomotives/{id}`
Возвращает данные о конкретном локомотиве.

### 4. Обновить данные локомотива
**PUT / PATCH** `/api/locomotives/{id}`
- `model` (string, optional): Модель (серия) локомотива
- `depot` (string, optional): Депо приписки

### 5. Удалить локомотив
**DELETE** `/api/locomotives/{id}`
Удаляет запись о локомотиве из базы данных.

## Телеметрия и Статистика (Telemetry)

### 1. Текущие геопозиции (Карта)
**GET** `/api/map/positions`
Возвращает актуальные позиции по всем локомотивам. Для каждого объекта возвращается: `id`, `series`, `lat`, `lng`, `speed` и `status`.

### 2. Последняя история телеметрии
**GET** `/api/locomotives/{id}/stats`
Возвращает 50 последних записей телеметрии для выбранного локомотива, отсортированных от новых к старым.

### 3. Аналитика за период
**GET** `/api/telemetry/analytics`
Возвращает исторические (агрегированные) данные для построения графиков.

Параметры запроса:
- `locomotive_id` (string, required): Идентификатор локомотива.
- `start_date` (date, required): Начало периода.
- `end_date` (date, required): Конец периода.
- `step` (integer, optional): Шаг агрегации (усреднения) в секундах. Если передан, данные усредняются за указанный интервал.

## WebSocket Эндпоинты (Endpoints)

Управление ссылками на WebSocket серверы телеметрии (для настроек подключения).

### 1. Получить список WS-серверов
**GET** `/api/endpoints`

### 2. Добавить WS-сервер
**POST** `/api/endpoints`
- `url` (string, required): Ссылка для подключения (например, `ws://localhost:8080/ws`)
- `is_active` (boolean, optional): Статус активности
- `description` (string, optional): Описание

### 3. Получить информацию о сервере
**GET** `/api/endpoints/{id}`

### 4. Обновить сервер
**PUT / PATCH** `/api/endpoints/{id}`
- Обновляет поля `url`, `is_active` и `description`.

### 5. Удалить сервер
**DELETE** `/api/endpoints/{id}`
